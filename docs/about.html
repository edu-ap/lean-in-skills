About

What this project exists for

lean-in-skills is the canonical home of a specific design-philosophy position: that the slice of AI-system guardrails which is decidable, high-stakes, and verifiable at the right moment is worth formalising in Lean 4 with a verifier-in-the-loop. The rest stays in prose where it belongs.

The work was built inside a working AI-augmented engineering practice and progressively extracted as an open source artefact. The runtime here is what skill-specific schemas share; per-domain applied schemas live with the skills that own them in their respective downstream repositories.

The author

Eduardo Aguilar Pelaez — engineer, polymath, formal-methods practitioner. Currently CTO of Legal Engine (UK SME voice-first legal AI).

This project is independent of Legal Engine's commercial work. Domain-specific applied schemas (legal-AI ones in particular) live with the skills that own them, not here.

Find Eduardo at github.com/edu-ap.

Roadmap

Items planned but not yet shipped (in rough priority order):

  • Cycle detection for typed composition graphs (Pilot 1 extension; non-Mathlib fuel-bounded recursion is feasible)
  • A worked example schema for a generic verdict-style skill output (GREEN / YELLOW / RED rating with required-fields-per-rating invariants)
  • A longer-form technical write-up: "Design-time vs runtime composability — when does each earn its keep?" (TODO; not yet authored)
  • bundle.py exporter for self-contained skill + runtime tarballs (for sharing skills to machines without the runtime installed)
  • More pilots: state-machine verifiers for posting lifecycles, allowlist predicates for security-critical sub-processor lists, schema invariants on PRD-style requirement trackers

Open to contributions on any of these. See GitHub issues to discuss.

How to engage

  • Use it: the pattern is Apache-2.0 licensed. Fork, adapt, ship.
  • Critique it: open an issue describing the failure case. The first-run-catches-real-bugs pattern works both ways.
  • Extend it: add a pilot for a new schema type. The shape is highly repeatable from the existing pilots.
  • Cite it: if it's useful in your work, a citation back to the repo is appreciated but not required.
