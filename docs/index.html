Where Lean 4 stops being type-theatre

Typed schemas plus a verifier-in-the-loop for AI skill outputs. Pragmatic formal methods for AI systems, not the comprehensive kind.

Lean replaces prose for the slice that is decidable, high-stakes, has a verifier in the loop, survives author absence, and has a read-to-verify ratio that justifies the syntax tax.
Read the thesis View on GitHub

What this is

A small Lean 4 library and a methodology for the narrow slice where formal methods earn their keep in AI-system design.

Three conditions, all required:

  • The skill produces a structured output (JSON, frontmatter, a typed record).
  • The output has invariants you care about (required fields, consistent enums, no dead references, evidence-or-explicit-self-attestation for any "done" state).
  • A verifier can actually run against real outputs at the right moment (pre-commit hook, post-skill-invocation check, CI step).

For that slice, a typed Lean schema plus a by decide proof beats prose guardrails. Outside that slice, prose wins. This project is the discipline of telling them apart.

Who this is for

Engineering leaders building production AI systems Formal-methods practitioners curious about AI applications AI infrastructure builders The design-time-composability camp Anyone fighting LLM non-determinism at scale

If you build agent systems with five tools, prose guardrails are fine and this overlay is type-theatre. If you build with fifty and care about safety-critical outputs, the syntax tax pays back.

Architecture at a glance

┌────────────────────────────────────────────────────────────────┐ │ Authoring surface AI / human writes structured │ │ output (JSON, YAML, MD) │ └────────────────────────────────────────────────────────────────┘ │ │ Ingestion (Python → Lean instance) ▼ ┌────────────────────────────────────────────────────────────────┐ │ Lean representation Typed value with structural │ │ invariants baked into types │ └────────────────────────────────────────────────────────────────┘ │ │ Verifier (lake build + by decide) ▼ ┌────────────────────────────────────────────────────────────────┐ │ Verdict Pass/fail; failing theorem names │ │ what's wrong │ └────────────────────────────────────────────────────────────────┘

Per-skill schemas live in each skill's folder and travel with it. The runtime (lakefile, toolchain pin, generic infrastructure) is shared. Bundles are portable; verifiers are auditable; non-conformant output is mechanically rejected at save time.
